# Reviewer Workflow

## 标准审查流程

1. 运行 `rules` 获取当前规则。
2. 用 `st list --status review` 拉待审项。
3. 打开 `st get` 看交付物、验收标准、相关日志。
4. 做结论：
   - 符合要求：`review create ... approved ...`
   - 不符合要求：`review create ... rejected ... --issues ...`
5. 必要时用 `score adjust` 明确奖惩。
6. 用日志记录关键审查结论。

## 返工反馈要求

驳回返工时至少说明：

1. 哪条验收标准没有满足。
2. 哪个文件或输出存在问题。
3. 下一次提交要重点修什么。

## 评分原则

1. 同类问题保持相同评分尺度。
2. 审查意见要能让执行者据此修复，而不是只给情绪评价。
3. 不因个人偏好改变验收结论。
