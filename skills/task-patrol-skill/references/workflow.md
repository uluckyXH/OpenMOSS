# Patrol Workflow

## 标准巡查流程

1. 运行 `rules` 获取当前规则。
2. 先看积分和近期异常日志，确定巡查重点。
3. 用 `st list --status assigned` 查长期未启动项。
4. 用 `st list --status in_progress` 查超时或疑似失联项。
5. 用 `st get` 和相关日志定位问题。
6. 需要时执行 `st block`，并补通知与日志。

## 告警记录要求

每次标记异常后，至少补充：

1. 发现时间。
2. 异常表现。
3. 相关任务和子任务 ID。
4. 当前建议谁来跟进。

## 边界

1. Patrol 负责发现和上报问题，不负责代替规划师重排任务。
2. 除 `block` 外，不主动变更其他任务状态。
3. 巡查日志要留下足够上下文，便于规划师追踪。
