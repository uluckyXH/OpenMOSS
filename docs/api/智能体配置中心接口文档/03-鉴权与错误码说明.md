# 鉴权与错误码说明

> 最后同步：2026-04-03
> 对应代码：`app/auth/dependencies.py`

## 1. 鉴权方式总览

当前智能体配置中心相关接口涉及三种鉴权方式。

### 1.1 管理员鉴权

请求头：

```http
X-Admin-Token: <admin_token>
```

说明：

- 适用于管理端接口
- Token 由 `/api/admin/login` 登录后产生
- 当前存在于内存中，服务重启后失效

失败时当前返回：

```json
{
  "detail": "管理员验证失败，请重新登录"
}
```

状态码：`403`

### 1.2 Agent Bearer 鉴权

请求头：

```http
Authorization: Bearer <api_key>
```

说明：

- 适用于 Agent 侧运行时接口
- 当前直接用运行态 `agent.api_key` 做明文比对

失败场景与返回：

| 场景 | 状态码 | 返回 |
|---|---|---|
| `Authorization` 不是 `Bearer <api_key>` 格式 | `401` | `Authorization 格式错误，需要 Bearer <api_key>` |
| API Key 无效 | `401` | `无效的 API Key` |
| Agent 已禁用 | `403` | `Agent 已禁用，无法操作` |

### 1.3 Agent 注册令牌

请求头：

```http
X-Registration-Token: <registration_token>
```

说明：

- 仅用于 `POST /api/agents/register`
- 当前仍为旧运行态注册链路
- 后续配置中心主线会逐步转向 Bootstrap Token

失败场景与返回：

| 场景 | 状态码 | 返回 |
|---|---|---|
| 自注册关闭 | `403` | `Agent 自注册已关闭，请联系管理员创建` |
| 注册令牌无效 | `403` | `注册令牌无效` |

## 2. 当前错误码口径说明

当前代码里有两类错误来源：

1. 路由中显式抛出的 `HTTPException`
2. service 层抛出的 `ValueError`，再由：
   - 路由局部转换为 `400/404`
   - 或 [main.py](/Volumes/MacSSD/Repositories/OpenMOSS/app/main.py) 中的全局异常处理统一转成 `400`

因此当前会出现一个现实情况：

- 有些“资源不存在”会返回 `404`
- 有些“资源不存在”会因为走了 `ValueError` 全局处理，返回 `400`

这不是文档写错，而是当前实现口径还没有完全统一。接口文档已经按**当前实际行为**写明。

## 3. 常见错误码清单

| 状态码 | 场景 | 说明 |
|---|---|---|
| `400` | 业务错误 | 例如 slug 冲突、角色模板不存在、部分资源不存在 |
| `401` | Agent 鉴权失败 | Bearer 格式错误、API Key 无效 |
| `403` | 权限不足 / 鉴权失败 | 管理员 token 无效、Agent 已禁用、自注册关闭 |
| `404` | 资源未命中 | 某些接口显式抛出的不存在错误 |
| `422` | 校验失败 | 缺 Header、Body 字段缺失、字段类型不合法 |

## 4. 脱敏字段说明

当前接口里以下字段不会回显明文：

### 4.1 宿主配置

响应字段：

- `host_config_payload_masked`

来源：

- 请求体中的 `host_config_payload`
- 落库后当前先原样存储
- 接口返回时通过 `_mask()` 脱敏

### 4.2 宿主通讯渠道配置

响应字段：

- `config_payload_masked`

来源：

- 请求体中的 `config_payload`
- 接口返回时进行脱敏

### 4.3 脱敏规则

当前 `_mask()` 规则是：

- 为空：返回 `null`
- 长度 `<= 8`：全部替换成 `*`
- 长度 `> 8`：保留前 `4` 位和后 `2` 位，中间替换为 `***`

示例：

| 原值 | 返回值 |
|---|---|
| `abcdef` | `******` |
| `1234567890` | `1234***90` |

## 5. 建议

- 联调时不要假设所有“资源不存在”都返回 `404`，要按当前接口文档逐个对接。
- 生产使用时，管理员接口和 Agent 接口不要混用鉴权头。
- 如果后面前端联调需要统一错误口径，建议单独起一轮“错误码收口”改造，而不是在业务开发过程中零散修。
